# Fix: Duplicate React key `undefinedscheduled` in CartPageContent

## Root Cause

**File:** `src/features/cart/components/CartSection.tsx:142`

```tsx
key={item.product_id + deliveryType}
```

When `item.product_id` is `undefined`, JS string concatenation produces `"undefinedscheduled"`. Multiple items with undefined `product_id` share this key.

The `product_id` comes from `CartPageContent.tsx:159` where server cart items are mapped:
```tsx
product_id: item.product.id,
```
If the API returns items where `item.product` is missing/null (e.g., deleted products), `product_id` becomes `undefined`.

## Fix (2 changes)

### 1. `src/features/cart/components/CartPageContent.tsx` (~line 158)
Filter out server cart items with missing/invalid product data before mapping:
```tsx
const items: GuestCartItem[] = (cart?.items ?? [])
  .filter((item) => item.product?.id != null)
  .map((item) => ({ ... }));
```

### 2. `src/features/cart/components/CartSection.tsx` (line 142)
Use `cartItemId` (unique per cart entry) as the key instead of `product_id + deliveryType`:
```tsx
key={item.cartItemId ?? `${item.product_id}-${item.deliveryType}`}
```

This is a two-layer defense: filter bad data at the source, and use a robust key as a fallback.
